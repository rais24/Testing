Fitly Style Guide

This style guide serves as a record for Fitly's visual design requirements on the web. Please consult it during development to maintain visual consistency across various devices during the development cycle.

If you don't see design specifications called out explicitly, just inspect the element with your browser's dev tools. If an element is missing entirely, please contact Jay Sylvester to have it added.

Color Scheme

These colors are the foundation for elements that should attract the user's attention: navigational elements, section headers, form buttons, etc. They should serve as the basis for any gradients as well.

Green should be used for navigation (site header, links, and buttons) and headers, orange for accents, and red for errors and warnings. Gradients should be reserved for the site header and buttons. Section headers should be flat.

  • rgb(129, 218, 27)
  • rgb(105, 184, 20)
  • rgb(21, 147, 38)
  • rgb(243, 160, 18)
  • rgb(224, 77, 36)

Headers and Body Copy

All headers are in Asap Bold typeface. First-level header styles (<h1>) will vary from page to page depending on content, but the default is 24px, dark grey (#6f6f6f). Second-level headers (<h2>) are 20px, white, with a dark green background that extends to the edges of the viewport.

Default body copy is Helvetica/Arial/Sans Serif, 14px, black, with a line height of 1.5 (no units). Links within body copy should be dark green, bold, and underlined. The underline should disappear on hover.
